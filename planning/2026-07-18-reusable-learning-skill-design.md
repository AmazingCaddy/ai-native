# 可复用学习 Skill 设计

## 目标

创建全局 Codex skill `run-learning-program`，使任何仓库都能启动或继续一套可追溯的学习流程，并发布为 GitHub Pages 学习站。

## 能力

- 开始新方向时收集学习目标、基础、时间和周期。
- 每天开始前建立问题清单和主线游标。
- 原样保存所有知识问答，无论正确与否。
- 首轮解释后保持当前题为 `待追问`；只有学习者明确继续才移动主线游标。
- 使用全局编号管理跨主题、跨天讨论。
- 将稳定结论整理到唯一正式资料区 `docs/review/`。
- 为时效性知识保存一手来源和核验日期。
- 检测并复用现有文档框架；配置 GitHub Pages 发布。
- 默认允许发布完整学习过程，但发布前必须执行密钥、隐私和机密信息检查。

## 分发

技能安装到 `~/.codex/skills/run-learning-program/`，从而在不同仓库中自动发现。技能包含非破坏性初始化脚本、可复制 Markdown 模板和 GitHub Pages 发布规范。

## 验证

- 使用 skill-creator 的结构验证器检查 skill 元数据。
- 在临时空仓库运行初始化脚本两次，确认首次创建且再次运行不覆盖记录。
- 对生成目录执行内容和占位符检查。

## 2026-07-18 流程修订

真实使用发现“解释完成后立即前移游标”会错误假设学习者没有追问。问题生命周期修订为：`待讨论 → 待追问 → 用户明确继续 → 已完成 → 下一题`。
