# Day 1：Floway 直连三模型对比报告

日期：2026-07-19

Provider：Floway，OpenAI-compatible Responses API 直连

模型：`gpt-5.6-terra`、`gpt-5-mini`、`claude-sonnet-4-5`

实验规模：4 类任务 × 3 个模型 × 2 次重复，共 24 次真实调用

## 自动指标

| 模型 | 成功 | 平均延迟 | 平均输入 Token | 平均输出 Token | 输出 Token 总计 |
| --- | ---: | ---: | ---: | ---: | ---: |
| `gpt-5.6-terra` | 8/8 | 3,284 ms | 125 | 201 | 1,607 |
| `claude-sonnet-4-5` | 8/8 | 7,014 ms | 175 | 319 | 2,548 |
| `gpt-5-mini` | 8/8 | 10,294 ms | 125 | 854 | 6,833 |

Token 为 Floway Responses API 返回的用量，可能包含模型内部推理 Token；不能直接等同于可见文本长度。由于没有纳入各模型价格，本报告不比较实际货币成本。

## 逐任务人工观察

| 任务 | `gpt-5.6-terra` | `gpt-5-mini` | `claude-sonnet-4-5` |
| --- | --- | --- | --- |
| 行动项提取 | 第一次多提上线目标，第二次准确；两次均为可直接解析 JSON | 两次都多提上线目标和 CSV 限制，边界控制最弱 | 两次都多提上线目标，并添加 Markdown 围栏，无法直接按纯 JSON 解析 |
| 代码解释 | 两次核心分析正确，但分别 348、482 字，违反 300 字限制 | 两次核心分析正确，但分别 370、332 字，违反限制 | 两次都超长；第二次错误声称最后一次失败后不会等待，而原代码实际上仍会等待 |
| 资料问答 | 两次均正确引用“最多 5 天”，简洁 | 两次均正确，但报告输出 Token 明显更高 | 两次均正确，引用清晰 |
| 澄清问题 | 两次均为 8 题，覆盖用户、任务、数据、风险和指标，整体最稳 | 内容覆盖完整，但附带大量解释和继续给方案的表述，不够严格遵守“只提出问题” | 覆盖较完整；第一次结尾额外提出一个问题，存在超过 8 题的边界争议 |

所有模型的代码解释都违反 300 字限制，说明自然语言长度要求仍需应用侧确定性校验，不能仅依靠 Prompt。

## 结论

在当前小型强约束烟雾集上，`gpt-5.6-terra` 是最合适的默认选择：延迟最低、输出 Token 最少、格式遵循最好，且行动项提取第二次达到人工参考答案。它仍会出现一次业务误提取，不能替代 Schema、证据校验和回归评测。

`claude-sonnet-4-5` 的事实问答和自然语言解释可用性较好，但当前输出习惯不适合依赖纯 JSON 或严格长度的接口；应使用 Provider 支持的 Structured Outputs 和应用校验。

`gpt-5-mini` 在本实验中没有体现速度或 Token 优势，并且行动项边界控制最弱。由于未比较价格和更多任务，不能据此断言它在其他场景没有成本优势。

## 边界

- 只有 4 类合成任务、每类 2 次，样本很小。
- 行动项任务的 Prompt 没有使用 Day 2 最终 Structured Outputs，因此该结果衡量的是基础指令遵循，不代表生产接口最终表现。
- 模型均通过同一 Floway 兼容层调用，结果可能同时受到代理实现影响。
- 本实验适合选择下一轮候选和发现失败模式，不足以做通用模型排行榜。
