# 学习记录规则

14 天课程每天使用一份独立学习记录：`day-01-notes.md` 至 `day-14-notes.md`。

本课程采用双轨记录：

- `docs/learning/` 保存真实学习过程，包括原始回答和纠错轨迹。
- `docs/review/` 保存整理后的图表、精选问题和相对标准答案。

同时使用四层回溯系统管理展开讨论：每日问题清单、全局讨论索引、独立讨论线程和整理后的复习材料。

## 必须记录的内容

凡是用于检查知识理解的问答，无论回答正确、错误、不完整或“不知道”，都必须记录：

1. 问题
2. 学习者的原始回答，不润色、不覆盖
3. 判断：正确、部分正确、待修正或待回答
4. 标准答案与解释
5. 易错点
6. 一句话记忆
7. 复习状态和日期

实践操作、环境信息和普通流程确认不强制记录。

## 记录原则

- 按问答发生顺序编号。
- 初次回答永久保留，后续修正另行追加。
- 不把“看懂了”视为掌握；通过复述、辨析或应用题复习。
- 回答不正确时先解释差距，再安排一道相邻问题验证。
- 每天结束时标记需要次日复习的内容。
- 第 3、7、14 天优先复习仍未掌握的问题。
- 每轮知识问答讲解完成后，同步更新对应的复习材料。
- 尚未回答的问题不提前写入复习材料，避免泄露练习答案。
- 每天开始前创建 `day-XX-agenda.md`，列出主问题、状态和返回位置。
- 展开讨论使用 `DISC-NNN` 编号登记到 `docs/discussions/`，跨天时继续同一线程。
- 讨论结束后按照问题清单回到最近的未完成主问题。

## 问题生命周期

```text
待讨论 → 待追问 → 用户明确继续 → 已完成 → 下一题
```

- 给出首轮解释后，只把当前问题标记为 `待追问`，主线游标保持不变。
- 用户继续追问时，始终停留在当前问题；必要时创建独立讨论线程，但不激活下一题。
- 只有用户明确表示“继续”“下一题”“没有追问了”等推进意图，才将当前问题标记为 `已完成` 并移动游标。
- 不通过预先提出下一题来暗示当前讨论已经结束。

## 复习状态

- `待回答`：问题已经提出，尚未作答。
- `待复习`：已经讲解，但还没有再次验证。
- `已掌握`：能够脱离答案正确复述或应用。
- `需加强`：复习后仍有关键错误。
