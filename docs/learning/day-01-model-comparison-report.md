# Day 1 模型能力对比实验报告

日期：2026-07-18

> 本页是实验记录，不是跨任务的模型排行榜。经过校正的复习结论见 [Day 1 正式复习资料](../review/day-01-review.md#day-1-烟雾实验结果)。

## 实验范围

- Provider：Floway，经 Codex CLI 间接调用 GitHub Copilot 模型。
- 模型：`gpt-5.6-sol`、`claude-opus-4-7`。
- 任务：行动项提取、代码解释、基于资料的事实问答、需求澄清。
- 重复次数：每个模型、每个任务执行两次，共 16 次。
- 控制条件：相同 Codex Agent 外壳、相同 Prompt、相同输入和只读空目录。
- 原始证据：`apps/assistant/eval/model-comparison/results/2026-07-18T15-28-02-226Z.jsonl`。

## 人工评分

| 任务 | GPT 两次结果 | Claude 两次结果 | 观察 |
| --- | --- | --- | --- |
| 行动项提取 | 正确；裸 JSON；可直接解析 | 均多提取 1 项，并添加 Markdown 代码围栏 | GPT 更严格遵守行动项边界和输出格式 |
| 代码解释 | 内容正确；274/264 字 | 内容正确；764/654 字 | Claude 两次均违反“不超过 300 字” |
| 事实问答 | 正确引用“最多 5 天” | 正确引用“最多 5 天” | 两者均满足标准 |
| 需求澄清 | 8 个问题，覆盖要求，较简洁 | 8 个问题，覆盖要求，较详细 | 两者均满足标准，风格不同 |

按“同时满足全部显式标准”计算，GPT 为 8/8，Claude 为 4/8。Claude 的内容并非普遍错误；主要失败集中在过度提取、代码围栏和字数约束。

## 失败案例

会议原文写的是“林岚确认批量导入功能仍以 8 月 15 日上线为目标”。人工标注没有把这句话视为新的行动项。Claude 两次都额外输出：

```json
{"task":"确认批量导入功能上线目标","owner":"林岚","dueDate":"2026-08-15"}
```

该内容有原文依据，但把“确认既有目标”解释成“待执行行动项”，属于边界判断错误，而不是无来源幻觉。

代码解释任务要求不超过 300 字。GPT 两次分别为 274 和 264 字；Claude 两次分别为 764 和 654 字。Claude 的技术解释正确且更详细，但不满足明确长度约束。

## 结论边界

本实验支持以下有限结论：在当前四个小型任务中，GPT 的格式遵循、简洁性和平均延迟更好；两个模型的重复输出都很稳定。

本实验不支持“GPT 的一般能力强于 Claude”。样本数量少、任务偏简单，并且测量的是模型在 Codex Agent 外壳中的表现。真实选型仍需要覆盖业务分布和多个难度等级的 Challenge Set。
