## 核心内容

### 按任务选择能力

- OCR：把图像中的字形转换成文本
- 文档解析：恢复页、段落、表格和标题等结构
- 视觉理解：识别物体、布局、图表和关系
- 语音识别：把音频转换为带时间信息的文本
- 多模态推理：联合文本和视觉或音频信息完成任务

先使用确定性解析器提取可可靠获得的结构，再让模型处理模糊语义。不要让模型重新发明文件校验、病毒扫描或权限控制。

### 输入链路

1. 校验 MIME、扩展名、大小和页数
2. 保存原始文件或安全的临时引用
3. 解析、转码或生成页面图片
4. 调用模型或专用识别服务
5. 用 Schema 校验提取结果
6. 保存来源位置、页码、时间戳和置信信息
7. 必要时进入人工校对

### 评测维度

- 字段精度和遗漏率
- 表格行列是否错位
- 页码、时间戳和来源是否可追溯
- 低质量、旋转、遮挡或噪声输入的处理
- 单文件成本、延迟和失败恢复

## 动手实验：专用解析与多模态模型

选择 5 个有代表性的输入，例如截图、扫描 PDF、架构图或会议音频。

先用传统解析器、OCR 或语音识别得到基线，再用多模态模型完成同一提取任务。比较精确字段、复杂布局理解、延迟、成本和失败模式。

## 主线项目任务

从以下能力中至少完成一个：

- 上传图片或 PDF 页面，提取标题、摘要和行动项
- 上传会议音频，转写后进入 Day 2 行动项提取流程
- 上传架构图，生成结构化组件和连接列表

实现时必须：

1. 限制文件类型和大小。
2. 保存来源位置或时间戳。
3. 对提取结果执行 Schema 校验。
4. 对失败、超时和不支持格式提供清晰状态。
5. 文件中的文字不能改变系统指令或工具权限。
6. 用 5 个样例形成回归评测。
