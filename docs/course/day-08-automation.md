## 核心内容

### 先选流程，再选工具

候选流程至少满足两个条件：频繁发生、输入较稳定、输出可检查、失败可恢复、节省时间明显。优先自动化低风险的资料整理和草稿生成，不要从自动发送、删除或批准开始。

### 工作流状态示例

```text
RECEIVED -> EXTRACTED -> VALIDATED -> WAITING_APPROVAL
         -> NEEDS_INPUT             -> EXECUTING -> COMPLETED
                                      `----------> FAILED
```

每个状态应能回答：进入条件、保存的数据、可执行动作、重试策略、超时处理和责任人。

### 模型节点的正确位置

- 适合：分类、摘要、字段提取、候选建议、模糊匹配
- 不适合：权限判定、金额计算、唯一性、事务一致性、最终审批
- 模型输出必须经过 Schema 和业务规则，再驱动下一步

## 动手实验：在任意步骤崩溃

对自动化流程人为注入以下故障：

- 模型超时
- 输出 Schema 无效
- 工具暂时不可用
- 用户重复提交
- 审批后进程退出

重启流程，确认不会丢失状态、重复执行写操作或绕过审批。

## 主线项目任务

实现以下默认流程，或选择价值更高的真实流程：

1. 接收会议记录或转写文本。
2. 提取行动项。
3. 执行业务校验并标记缺失字段。
4. 生成任务草稿。
5. 等待用户逐项批准或修改。
6. 调用任务工具。
7. 输出成功、失败和待处理摘要。

额外要求：

- 每次运行有唯一 ID 和幂等键
- 状态持久化，进程重启后可恢复
- 每一步有开始、完成和失败记录
- 模型和工具重试分别设置上限
- 记录人工原耗时和自动化后耗时
