## 核心内容

### 模型网关职责

- 从配置读取模型和提供商
- 统一超时、取消、重试和错误映射
- 记录请求 ID、Prompt 版本、延迟和 Token
- 支持普通响应和流式响应
- 隔离提供商 SDK 对业务代码的影响
- 为测试提供 Fake 或 Stub 实现

### 重试原则

- 网络瞬时错误、部分限流和部分服务端错误可以有限重试
- 参数错误、权限错误、内容不符合业务要求不能盲目重试
- 使用指数退避和随机抖动
- 写操作必须考虑幂等；模型生成请求也要设置总预算
- 用户取消后停止下游调用，不继续消耗 Token

### 模型选择维度

- 任务成功率和稳定性
- 输入模态与上下文需求
- 输出速度和 P95 延迟
- 单次任务成本而非单 Token 价格
- 数据处理和区域要求
- 工具调用、结构化输出等能力

## 动手实验：观察真实请求

发送一次非流式和一次流式请求，记录：

- 首 Token 时间
- 总完成时间
- 输入和输出 Token
- 流式事件种类
- 用户中途取消后的行为
- 提供商返回的请求标识

然后制造超时、无效模型、无效凭据和限流中的至少两种错误，确认应用错误信息稳定且不会泄露凭据。

## 主线项目任务

1. 定义 `ModelClient` 或等价接口。
2. 实现一个真实提供商适配器和一个测试 Fake。
3. 模型通过环境变量配置，提供安全默认值或启动校验。
4. 为聊天或回答接口增加流式输出。
5. 支持取消请求和全局超时。
6. 记录模型、Prompt 版本、延迟、Token 和结果状态。
7. 为重试次数、最大输出和单次请求预算设置上限。
8. 添加超时、解析失败和提供商错误测试。
