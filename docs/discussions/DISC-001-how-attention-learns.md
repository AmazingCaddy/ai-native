## 原始追问

> 你说得有点抽象，一开始的结论是怎么来的，人来理解是这样的，计算机为什么也能这样理解
>
> “它”关注“书” 较低
> “它”关注“桌子” 较高
> “它”关注“稳固” 属性线索

## 为什么值得独立讨论

这个问题连接了模型训练和模型推理两个阶段，也涉及“计算机是否真的理解”的边界。它需要区分模型参数与每次输入产生的 Attention 权重，因此单独追踪。

## 讨论记录

### 2026-07-18：权重不是人工写入的规则

模型中有两种容易混淆的“权重”：

1. **模型参数**：Embedding 以及 `WQ`、`WK`、`WV` 等矩阵。它们最初近似随机，通过训练和反向传播逐步更新，训练完成后通常固定。
2. **Attention 权重**：模型处理某次具体输入时，使用已训练参数现场计算出的相关性分布。换一句话，数值也会改变。

训练时，模型在海量文本上反复预测 Token。预测错误会产生 Loss，反向传播根据 Loss 调整参数。经过大量样本后，能帮助预测的语言关系会被编码进分布式参数中。没有工程师手写“桌子比书更稳固”这条规则。

使用时，每个 Token 表示会通过已学习矩阵转换成 Query、Key 和 Value。Query 与各 Key 的点积经过缩放和 Softmax，得到总和为 1 的 Attention 权重；这些权重再乘以对应 Value 并求和。

### 对先前示意图的修正

先前把“稳固”直接连向“它的表示”并不适合描述因果语言模型。在因果 Attention 中，“它”所在位置不能查看后面的“很稳固”。但当模型在完整句子之后生成回答时，**回答位置**可以回看整句，因此可以同时利用“桌子”“它”和“稳固”等信息。

双向编码器的 Token 可以查看左右上下文；因果生成模型的当前位置只能查看它之前的 Token。两种架构不能混画成同一条信息流。

## 阶段性结论

- Attention 不是预先写死的语义规则。
- 模型参数通过预测任务、Loss 和反向传播从数据中学习。
- Attention 权重由训练后的参数和当前输入共同决定，每次输入都重新计算。
- “理解”在这里表示模型学到了对预测有用的统计结构，不等于拥有人类经验或主观理解。
- 单个 Attention 头或一组权重不能充当模型完整推理过程的证明。
